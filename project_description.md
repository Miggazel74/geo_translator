# Проект: Офлайн-переводчик (Грузинский <-> Русский)

## Описание проекта
Создание мобильного и настольного приложения для двустороннего перевода голоса и текста, работающего полностью без интернета. Проект оптимизирован для использования в Грузии на устройствах с архитектурой ARM (смартфоны) и Windows.

## Текущий стек технологий
1. **Распознавание речи (STT):** OpenAI Whisper (модель small) — обеспечивает высокую точность понимания живой речи.
2. **Перевод текста (NMT):** Facebook NLLB-200 (No Language Left Behind) — современная модель, специально обученная для качественного перевода менее распространенных языков, включая грузинский (kat_Geor).
3. **Синтез речи (TTS):** Библиотека TTS_ka — используется для озвучивания перевода на грузинский язык.
4. **Интерфейс и логика:** Python 3 с использованием библиотек SoundDevice и SpeechRecognition для работы с микрофоном.

## История разработки
- **Этап 1 (Windows):** Скрипт успешно протестирован на ноутбуке. Настроена загрузка моделей и базовая логика диалога.
- **Этап 2 (Android/Termux):** Начата миграция в среду Termux-Ubuntu на смартфоне.
- **Текущая задача:** Исправление ошибок совместимости библиотек PyTorch и Transformers в мобильной среде ARM64. Выполняется чистая установка официальных сборок нейросетевых движков для устранения ошибки "undefined symbol" в системных пакетах Ubuntu.

## Особенности реализации
- Весь перевод происходит локально на устройстве.
- Используется Python 3.13 в среде Ubuntu (PRoot) внутри Termux.
- Настройка PulseAudio для перехвата звука с микрофона смартфона внутри Linux-окружения.

## Технические детали установки (Android/Termux)

### Список основных библиотек (Python):
- **torch:** Движок нейросетей (устанавливается через pip3 с флагом --force-reinstall для исправления системных ошибок).
- **transformers, tokenizers, sentencepiece:** Инструменты для работы с моделями Hugging Face.
- **sounddevice, pyaudio:** Для захвата и вывода звука.
- **SpeechRecognition:** Оболочка для работы с микрофоном.
- **numpy:** Математические операции со звуковыми потоками.
- **TTS_ka:** Грузинский синтезатор речи.

### Системные зависимости и переменные:
- **Путь библиотек:** `/usr/local/lib/python3.13/dist-packages` (стандарт для pip в этой среде).
- **Переменные окружения:**
  - `export PULSE_SERVER=127.0.0.1` — для связи со звуковым сервером Android.
  - `unset LD_PRELOAD` — для предотвращения конфликтов библиотек Termux и Ubuntu.
- **Критические фиксы:**
  - Удаление системных пакетов `python3-torch` через `apt`, чтобы они не конфликтовали с «настоящим» torch из pip.
  - Использование `--break-system-packages`, так как в новых версиях Python в Ubuntu прямая установка через pip ограничена.

## Хроника решения проблем (Android/Termux)

1.  **Проблема 1: Ошибка "undefined symbol" в PyTorch.**
    - *Причина:* Системная сборка из репозитория Ubuntu (apt) оказалась несовместима с процессором телефона.
    - *Решение:* Полное удаление системных пакетов `python3-torch` и установка официальной сборки через pip.

2.  **Проблема 2: Попытка скачать 4 ГБ библиотек NVIDIA (CUDA).**
    - *Причина:* Пакет `torch` по умолчанию пытался скачать драйверы для ПК видеокарт.
    - *Решение:* Установка строго **CPU-версии** через `--index-url https://download.pytorch.org/whl/cpu`.

3.  **Проблема 3: Конфликт с системным пакетом networkx.**
    - *Причина:* `pip` не мог удалить файлы, установленные через `apt`.
    - *Решение:* Установка с флагом `--no-deps` для пропуска попыток переустановки системных зависимостей.
